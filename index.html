📘 Dashboard Help

ChatGPT Apps — Category × Capability Dashboard

Overview

This dashboard provides a structured, explorable view of ChatGPT-connected applications, organized by categories and capabilities.

  • Categories (e.g. Design, Productivity, Business)
  • Capabilities are organized into two levels:
    • Plugin capabilities: Interactive(API), Writes(API)
    • Connector capabilities: File Search(MCP), Sync

It helps users quickly answer questions like:

  • Which apps support Plugin capabilities?
  • Which apps support Connector capabilities?
  • What design tools support file search?
  • Which apps match a specific capability combination?

The dashboard supports four complementary views:

  • Matrix View — high-level comparison
  • List View — detailed, filterable results
  • Timeline View — chronological exploration
  • Microsoft View — dedicated Microsoft Copilot connectors catalog

Data Model

  • App name
  • Category (single)
  • Capabilities (one or more, organized by level)

Capability Levels

Capabilities are categorized into two levels to help you understand the type of integration:

  • Plugin capabilities — Enable interactive actions and writes through APIs:
    • Interactive(API): Apps that support interactive operations via API
    • Writes(API): Apps that can write or modify data via API
  • Connector capabilities — Enable data access and synchronization:
    • File Search(MCP): Apps that support file search through MCP (Model Context Protocol)
    • Sync: Apps that support data synchronization

Throughout the dashboard, capabilities are displayed with their level (e.g., "Interactive(API) (Plugin)") and are visually color-coded: blue for Plugin capabilities and green for Connector capabilities.

Matrix View (Overview)

Purpose: Visual comparison across categories and capabilities.

  • Columns → Categories
  • Rows → Capabilities
  • Cells → Apps matching category and capability
  • Heatmap shading → Darker cells indicate more apps

Interactions

  • Click a cell to see apps inside it
  • Filter visible rows or columns
  • Export visible apps as CSV

List View (Search & Discovery)

Purpose: Precise filtering and app discovery.

  • Scrollable list of apps
  • Shows app name, category, and capability chips

Timeline View (Chronological Exploration)

Purpose: Explore apps organized by their release date.

  • Apps grouped by year and month (newest first)
  • Visual timeline layout with year badges and month labels
  • Responsive grid showing app cards with release dates
  • Each card displays: app name, release date, category, and capabilities
  • Search box to filter apps by name or category
  • Great for tracking the evolution of the ChatGPT apps ecosystem

Interactions

  • Click any app card to view full details
  • Use search to find specific apps by name or category
  • Scroll through the timeline to explore apps across different time periods

Microsoft View (Connector Catalog)

Purpose: Comprehensive catalog of Microsoft Copilot connectors, including both indexed and federated integration types.

  • Unified table showing all Microsoft Copilot connectors (28 total)
  • Displays connector name, status, capabilities, scope, and type
  • Includes both native integrations (SharePoint, Outlook, Teams) and third-party connectors
  • Shows various connector types: Native Graph, Sync, File Search MCP, and Custom MCP
  • Searchable by connector name, status, capabilities, scope, or type

Column Descriptions:

  • Connector — Name of the Microsoft Copilot connector
  • Status — Current availability: GA (Generally Available), Public Preview, WIP (Work in Progress), Pilot, Exploratory, Discussed, Experimental, or Not started
  • Capabilities — Integration type: Native Graph, Sync, File Search MCP, or Custom MCP
  • Scope — What the connector can access (e.g., "Files, folders, permissions" or "Live file access, summaries, lookup")
  • Type — Description of why this connector matters (e.g., "Core M365 substrate; primary enterprise knowledge source")

Connector Types Explained:

  • Native Graph — Built-in Microsoft 365 services (SharePoint, Outlook, Teams)
  • Sync — Connectors that index and synchronize data from external sources into Microsoft Graph
  • File Search MCP — Federated connectors using Model Context Protocol for real-time, non-indexed access
  • Custom MCP — Customer-built MCP servers for specialized business systems

Interactions:

  • Use the search box to filter connectors by any column value
  • Browse the complete catalog of Microsoft Copilot integrations
  • Understand which connectors are available vs. in development
  • Compare indexed (Sync) vs. federated (MCP) connector approaches

Capability Query Builder

Filter apps using natural language or manual selectors.

Examples:

  • interactive
  • interactive and writes
  • sync but no file search
  • interactive api but no writes

The parser is case-insensitive and ignores spaces and punctuation (interactive(api) = interactive api).

Apply Button

  • Parses the text query (if present)
  • Applies the resulting filters

Clear Button

  • Resets text query
  • Clears include/exclude capabilities
  • Resets list results

Include Capabilities (AND)

Apps must have all selected capabilities.

Exclude Capabilities

Apps must have none of the selected capabilities.

Result Controls

  • Result count shows matching apps
  • Search within results filters by name, category, or notes

Export

Export visible apps (CSV) based on current filters and view.

Typical Use Cases

  • 🔍 Find compatible apps
  • 🧭 Explore categories
  • 📊 Gap analysis
  • 📁 Reporting

Tips & Best Practices

  • Use Matrix view for discovery
  • Use List view for precision
  • Use Timeline view for chronological exploration
  • Use Microsoft view to explore enterprise connector options
  • Combine Include + Exclude filters
  • Export after filtering

Known Limitations

  • Categories are single-valued per app
  • Capability matching is exact
  • Free-text search does not override capability filters
